# 作业：图文笔记社区 Web App

## 1. 你要做什么

做一个**图文笔记类社区**网站。用户注册后可以发"一段文字 + 多张图"的笔记，看别人的笔记，通过点赞 / 收藏 / 评论 / 关注 / 标签 / 搜索发现内容。

参考产品：**小红书**、**Pinterest**。形态：登录后首页是**双列瀑布流**的图片笔记，点进去能左右翻多张图。

---

## 2. 谁会用 · 怎么用

- 学生：发"今天看的书 + 几张摘抄"
- 摄影爱好者：发"周末拍的 9 张照片 + 一句话"
- 兴趣社群：用话题 `#手账` 把同好串起来

典型 30 秒使用流：
打开 → 瀑布流里看到一张图被吸引 → 点进去左右滑完 6 张图 → 收藏 → 关注作者 → 在评论区 @ 朋友 → 顺着 `#手账` 标签去看更多。

---

## 3. 功能清单

### 3.1 账户
- 邮箱 + 密码 注册 / 登录
- 昵称（1–20 字符）+ 头像（用 emoji 选即可，免去文件上传）
- 个人简介 bio（≤ 80 字）
- 登录后能修改昵称 / 头像 / bio

### 3.2 发笔记 / 删笔记
- 一段文字（≤ 500 字）+ **1 到 9 张图**（JPG / PNG / WebP，单张 ≤ 2 MB）
- 仅作者本人能删除自己的笔记；删除后评论 / 点赞历史保留（软删除）

### 3.3 点赞
- 每人对每条笔记**最多点一次**（DB 唯一约束保证）
- 点赞数实时显示，再点一次取消

### 3.4 收藏
- 收藏按钮与点赞**完全独立**的动作（一条笔记你可以只收藏不点赞，反之亦然）
- 个人主页有 "**我的收藏**" tab（仅自己可见）

### 3.5 评论
- 单层评论（不嵌套）
- ≤ 200 字
- 按时间倒序展示
- 仅作者本人能删除自己的评论

### 3.6 关注
- 关注 / 取关任意其他用户
- 不能关注自己
- 关注关系决定"关注流"里能看到谁的内容

### 3.7 信息流（两条）
- **首页流**：所有公开笔记，按时间倒序，**双列瀑布流**布局（masonry），图片**按真实纵横比**显示，**懒加载**，cursor 分页
- **关注流**：只看我关注的人的笔记，瀑布流同上

### 3.8 笔记详情页
- **图片轮播**：左右按钮 + 键盘 ← → 切图 + 底部小圆点指示当前张
- 完整正文（自动把 `#标签` 和 `@昵称` 渲染成链接）
- 点赞 / 收藏 / 评论按钮
- 评论列表 + 评论输入框

### 3.9 个人主页
- 头像 / 昵称 / bio / 关注数 / 粉丝数 / 笔记数
- 关注 / 取关按钮（对他人主页）
- 两个 tab：**笔记**（ta 发的）/ **收藏**（自己看自己的）

### 3.10 话题 / 标签
- 笔记正文里写 `#手账` → 系统自动识别并建立标签
- 点击标签进入**话题页**：展示带这个标签的所有笔记（瀑布流）
- 首页侧栏（或顶部）展示 "**近 7 天热门话题**" Top 10

### 3.11 搜索
- 顶部搜索框 → 结果页
- 两个 tab：**笔记**（搜正文）/ **用户**（搜昵称）
- 笔记搜索用 SQLite FTS5 全文索引；用户搜索用 LIKE
- 命中关键词在结果里**高亮**

### 3.12 @ 提及
- 笔记 / 评论里写 `@昵称` 自动解析为该用户的链接
- 触发 "提及通知"（见下）

### 3.13 通知中心
- 顶部铃铛 + 未读数**小红点**
- 通知类型：别人 **点赞** / **评论** / **关注** / **@** 了你
- `/notifications` 页查看完整列表，可"全部标记已读"
- 不做实时推送，前端每 30 秒轮询未读数即可

---

## 4. 数据模型（实体清单，字段细节学生自定）

- `users`
- `posts`
- `post_images`（一条笔记对应多张图，position 决定顺序）
- `likes` `bookmarks`（独立两张表）
- `comments`
- `follows`（多对多，约束 follower ≠ followee）
- `tags` + `post_tags`（多对多）
- `notifications`
- `mentions`
- FTS5 虚拟表索引 `posts.body`，用于搜索

## 5. 验收标准

学生在演示时需走通以下场景：

- [ ] 用 A、B 两个账号注册（不同 emoji 头像 + bio）
- [ ] A 发一条 6 张图 + 正文含 `#typescript #手账 @B` 的笔记
- [ ] 在 B 那边：顶部铃铛红点 +1，通知列表显示"A 提及了你"
- [ ] B 打开 A 的笔记，左右键切完 6 张图
- [ ] B 点赞 + 收藏；A 收到两条通知
- [ ] B 关注 A；A 收到第三条通知；B 的"关注流"出现这条笔记
- [ ] B 点击正文里的 `#手账` 进入话题页，能看到这条笔记
- [ ] 顶部搜索 "typescript" → 在"笔记"和"用户"两个 tab 都能看到合理结果
- [ ] B 进自己主页，"我的收藏" tab 能看到那条笔记
- [ ] B 编辑自己的 bio 后，A 在 B 主页能立即看到新 bio
- [ ] 自我关注请求 → 400；B 删 A 的笔记 → 403

## 6. 技术口径

- **不限技术栈**。建议参考：
  - 前端：React + Vite + TypeScript + Tailwind + 状态管理（Zustand / Redux 任一）
  - 后端：Node.js + Express + TypeScript + SQLite（better-sqlite3）
  - 测试：Vitest（前后端共用）+ supertest（API 集成）
- 认证：JWT + bcrypt
- 图片：本地磁盘 + UUID 重命名 + MIME 白名单
- 提交后能用一对 `npm run dev` 在 localhost 走完整流程

---

## 7. 规模与提交(可以参考final—project文件)

- **代码量目标**：源码 + 测试 **≈ 5,000 行**
- **必交文件**：
  - `SPEC.md`（设计规约：问题陈述 / 用户故事 / 用例 / 类图 / 顺序图 / 架构图 / 非功能）
  - `PLAN.md`（任务拆分与完成判定）
  - `README.md`（先决条件 / 安装 / 运行 / 端口 / 环境变量 / 目录结构）
  - `AGENT_LOG.md`（AI 协作过程时间线）
  - `REFLECTION.md`（反思报告 1500–2500 字，必须本人撰写）
  - 源代码 + 测试代码
  - CI 配置（push 自动跑 lint + typecheck + test）
- **测试要求**：后端单元覆盖率 ≥ 60%

## 8. 不在本作业范围内

明确不要求做（避免学生在这些地方耗时间）：

- 推荐算法（一切排序都按时间倒序；"热门话题"按近 7 天计数即可）
- 私信 / 群聊
- 视频 / GIF / 直播
- 屏蔽 / 举报 / 后台管理
- 多端推送 / WebSocket（通知靠轮询）
- Docker / 云部署
